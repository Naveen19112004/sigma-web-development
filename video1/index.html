1. Front-End vs Back-End

Front-End

  • The visible part of a website (what users interact with).
  • Built using HTML, CSS, JavaScript.
  • Runs inside the browser.
  • Handles layout, design, colors, animations, and interactions.

Back-End

  • The hidden part of a website (not visible to the user).
  • Handles logic, databases, authentication, data storage.
  • Runs on the server.
  • Ensures smooth performance without burdening the client side.

2. Client‚ÄìServer Architecture

Client

  • The user‚Äôs device (laptop/phone).
  • Sends requests like typing codewithharry.com into the browser.

Browser

  • Interprets front-end code.
  • Sends requests to the server.
  • Displays the final webpage.

Server

  • Stores website files and data.
  • Receives requests from the browser.
  • Sends back HTML, CSS, JS files and other resources.

Process Example

  1. User types example.com in the browser.
  2. Browser sends a request to the server.
  3. Server finds the required files/data.
  4. Server sends them back to the browser.
  5. Browser displays the webpage.

3. Front-End Core Technologies

HTML (HyperText Markup Language)

  • Structure of the webpage.
  • Defines headings, paragraphs, images, buttons, forms.

CSS (Cascading Style Sheets)

  • Styling of the webpage.
  • Controls colors, fonts, layout, spacing, responsiveness.

JavaScript

  • Adds interactivity and dynamic behavior.
  • Examples: sliders, dropdowns, animations, logic.

5. Back-End Overview

  • Runs on the server.
  • Not visible to the user.
  • Handles databases, APIs, authentication, and application logic.
  • Manages data flow between client and server.
