Block elements always take up the full line, even if their width is reduced. Reducing the width only shrinks the visible content area, but the remaining part of the line is still reserved for the block element. Inline elements cannot move into this space, because block-level elements force the next element to start on a new line.

Inline elements take only the required width and sit beside other inline elements, but they can never sit beside a block element in normal flow.

Margin Collapse: Margin collapse only happens between two block-level elements. Inline elements like <span> do not collapse margins, and their vertical margins (top/bottom) do not create visible gaps. That is why the 5px margin does not appear around the span in this example.

Universal Selector: * applies to all elements but has the lowest specificity. Element selectors (div, p, span) override it because they have higher specificity. CSS does not run line by line; it resolves conflicts based on specificity rules.
