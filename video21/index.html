CSS Specificity - Lecture 21 Notes

1. Source Order

If two rules have the same specificity, the rule written last wins. Example: If an element has classes bg1 and bg2, and bg2 is written later in CSS, its styles will override bg1.

2. Specificity Hierarchy

  • Inline styles → 1000
  • ID selectors → 100
  • Class, attribute, pseudo-class → 10
  • Element, pseudo-element → 1
  • Universal selector (*) → 0

Example: h1.something1 has specificity 1 (h1) + 10 (class) = 11.

3. CSS Origin

CSS comes from different sources:

  • Author styles – your CSS (highest priority)
  • User styles – user-defined browser settings
  • Browser default (User Agent stylesheet)

Author CSS overrides browser defaults.

4. !important Keyword

!important has the highest priority and overrides all other rules. It should be used only when absolutely necessary.

Summary

CSS specificity determines which style wins. Inline > ID > Class > Element > Universal. If specificity is equal, the last rule in the CSS file wins. The !important keyword overrides everything.

!important always overrides all CSS rules regardless of specificity. It has the highest priority in CSS. Specificity is checked only when two or more rules use !important. If only one rule has !important, it wins automatically over inline styles, IDs, classes, and all other selectors.
